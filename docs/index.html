History

The Problem

Classic project planning — and where it broke down

1

Developers

Break down work into tasks. No visibility into the overall plan or timeline.

2

Project Manager

Types tasks into MS Project, manually assigns developers — no validation.

3

Gantt Created

Developers have no MS Project license. They cannot review or give feedback.

4

Errors Surface Late

Missing dependencies and wrong assignments found mid-project, too late to fix cleanly.

5

Customer Impact

Two incidents where the delivered schedule was a full month too optimistic.

⚠

The feedback loop was broken. Developers could not see or validate the plan until it was far too late.

History

First Iteration: The Excel + qcd-tool Pipeline

The pipeline

  1. 📄 Excel Sheet Tasks, assignments & dependencies filled in by developers
  2. ⚙️ qcd-tool Reads Excel, generates intermediate Gantt chart for developer review
  3. 📊 Microsoft Project Final Gantt generated after conflict check with other projects
  4. 🎯 Jira Sprint CSV exported from qcd-tool imported into Jira to create the sprint

✓ What improved

  • Developers validated the plan before it was finalised
  • Dependencies checked in Excel before any import
  • Jira sprint auto-created from the same source

✕ Still missing

  • Five separate tools to install, license, and keep in sync
  • qcd-tool breaking after library updates
  • SMB shared drive required for all Excel files
  • jira's real-time progress monitoring makes it difficult to estimate delay
  • very difficult to update the plan after work was already logged

Key Differentiators

What makes Kassandra different?

01

API First

Everything runs on a REST API. The UI is just a client — so integrations are first-class citizens.

02

ER Diagram from Tests

ErDiagramTest is an integration test that auto-generates the Entity Relationship Diagram on every run.

03

Screenshots from Tests

GenerateScreenshots captures every screen in light and dark mode automatically — including this presentation's images.

04

Selenium UI Testing

Full UI test coverage using Selenium, including automated introduction videos generated by the test suite.

05

AI-Generated Videos

Intro videos produced with Selenium + Chatterbox TTS + ffmpeg. Browser audio captured and merged with video frames.

06

Auto National Holidays

Holidays computed automatically from each user's location. No manual entry, no missed bank holidays.

07

Min/Max Effort Guardrails

Estimate tasks with min/max bounds. Release date interpolated from actual progress against the confidence range.

Architecture

How projects are structured

Every entity flows from Products down to individual work-log entries

▶
⭐

▶
🏃
Sprint
Time-boxed iteration with a team
▶
📋
Task / Story
Work item with min/max estimate
▶
⏱️
Worklog
Actual hours logged per developer

📐 Min / Max Estimation

Each task carries an optimistic and a pessimistic effort bound. The Gantt buffer and release date confidence range are computed from both values automatically.

🔗 Dependency Graph

Stories and tasks can depend on each other across sprints. The scheduler respects the full dependency graph when building the Gantt chart — no manual reordering needed.

📅 Client-side IDs

Clients may generate entity IDs before sending to the server, reducing round trips during bulk planning sessions and sprint import workflows.

Security

OIDC Authentication

Keycloak-backed single sign-on — every API call is scoped to the logged-in user's permissions

🔐 Keycloak Integration

Full OIDC/OAuth2 single sign-on. Supports any Keycloak realm — or any OpenID-compatible identity provider.

👥 Group-based ACL

Access is controlled per-project via user groups. Members see only the products they are granted access to.

🛡 API Security

Every REST endpoint is secured with Spring Security + @PreAuthorize. The AI assistant also operates within the current user's OIDC token scope.

🧪 Testcontainers Keycloak

Integration tests spin up a real Keycloak container automatically — no manual setup needed for CI.

Technical

Tech Stack & Design Philosophy

Java + Spring Boot

Robust, well-supported backend ecosystem. ~98% of the codebase is Java.

Vaadin UI

Server-side Java UI framework. The frontend is just another API client — no JS framework needed.

Selenium

Full UI test coverage. Also powers the automated intro video generation pipeline.

Chatterbox TTS + ffmpeg

Browser audio output captured and merged with video frames to produce intro videos.

Docker (Phase 2)

Planned containerization for easy self-hosting. One command to start everything.

Stable Diffusion

Local AI image generation for user avatars. No cloud, no data leaves your server.
